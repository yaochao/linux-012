# Linux 0.12 On QEMU

[中文说明](./README.md)

This repository builds the two QEMU runtime images for Linux 0.12 from source and repo-owned manifests, then boots QEMU, reaches a shell, and verifies the guest by running `ls`.

The repo no longer stores third-party runtime images. The default runtime images are generated by the `rebuild/` workflow under `rebuild/out/images/`:

- `bootimage-0.12-hd`: compiled from patched Linux 0.12 kernel source
- `hdc-0.12.img`: assembled from repo-owned minimal userland sources, rootfs manifests, and a Minix v1 filesystem build flow

Supported host platforms:

- macOS arm64
- Ubuntu 22.04
- Windows 10

The acceptance target is a Linux 0.12 shell prompt at `[/usr/root]#` followed by a successful `ls`.

## Host Requirements

All hosts need:

- Python 3
- Docker
- QEMU

### macOS arm64

```sh
brew install qemu
```

### Ubuntu 22.04

```sh
sudo apt update
sudo apt install -y python3 qemu-system-x86 docker.io
```

### Windows 10

- Install Python 3
- Install Docker Desktop
- Install QEMU for Windows
- Ensure `qemu-system-i386.exe` is on `PATH`, or set `LINUX012_QEMU_BIN` to its full path

Check host dependencies with the wrapper for the current platform:

```sh
./scripts/bootstrap-host.sh
```

```powershell
.\scripts\bootstrap-host.ps1
```

```bat
scripts\bootstrap-host.cmd
```

## Commands

### Build the images explicitly

```sh
python3 rebuild/driver.py build
```

Outputs:

- `rebuild/out/images/bootimage-0.12-hd`
- `rebuild/out/images/hdc-0.12.img`

### macOS and Ubuntu 22.04

Interactive boot:

```sh
./scripts/run.sh
```

Automated verification:

```sh
./scripts/verify.sh
```

### Windows 10 PowerShell

Interactive boot:

```powershell
.\scripts\run.ps1
```

Automated verification:

```powershell
.\scripts\verify.ps1
```

### Windows 10 Command Prompt

Interactive boot:

```bat
scripts\run.cmd
```

Automated verification:

```bat
scripts\verify.cmd
```

`run` and `verify` automatically trigger `python3 rebuild/driver.py build` when the runtime images are missing.

Verification artifacts are written to `out/verify/`:

- `screen.txt`: final decoded VGA text screen
- `m.log`: QEMU monitor traffic
- `q.log`: QEMU stdout and stderr

Interactive runtime artifacts go to `out/run/`.

## What The Source Build Does

The `rebuild/` directory owns the full source-to-image flow:

- unpack `vendor/src/linux-0.12.tar.gz`
- apply repo-owned Linux 0.12 patches for modern toolchains and QEMU
- compile the Linux 0.12 boot image
- compile the repo-owned minimal Linux 0.12-compatible userland:
  - `/bin/sh`
  - `/bin/ls`
- create the directory tree, device nodes, and boot script from repo manifests
- build a Minix v1 root filesystem that Linux 0.12 can mount
- assemble `hdc-0.12.img`
- boot QEMU, enter the shell, and run `ls`

This phase does not try to recreate a full historical Linux 0.12 distribution. It builds the smallest system needed for this repository's runtime contract.

## Repo Layout

- `scripts/bootstrap-host.*`, `scripts/run.*`, `scripts/verify.*`: host-specific entrypoints
- `rebuild/driver.py`: source build, runtime, and verification entrypoint
- `rebuild/container/build_images.sh`: container-side image build script
- `rebuild/patches/linux-0.12/`: Linux 0.12 compatibility patches for QEMU and modern toolchains
- `rebuild/userland/`: repo-owned minimal Linux 0.12 userland sources
- `rebuild/rootfs/manifest/`: directory, device, and boot-file manifests
- `rebuild/tools/aout_pack.py`: packs generated ELF binaries into Linux 0.12 `ZMAGIC` a.out executables
- `rebuild/out/images/`: locally generated runtime images
- `tools/qemu_driver.py`: QEMU launch, VGA scraping, and key injection
- `vendor/src/linux-0.12.tar.gz`: upstream Linux 0.12 source archive

## Notes

- The boot image is shorter than 1.44MB. The driver pads it into `out/run/boot.img` or `out/verify/boot.img` before launch
- QEMU always starts with `-snapshot`, so repeated runs do not mutate `rebuild/out/images/hdc-0.12.img`
- macOS and Ubuntu 22.04 use a local Unix socket for the QEMU monitor and `-display curses` for interactive sessions
- Windows 10 uses a localhost TCP monitor and QEMU's default visible window for interactive sessions

## Asset Provenance

- `vendor/src/linux-0.12.tar.gz` comes from the kernel.org historic source archive
- Runtime images are no longer downloaded from third parties; they are built locally from repo-owned sources, patches, and manifests
