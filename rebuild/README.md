# Linux 0.12 Rebuild Pipeline

`rebuild/` owns the source-to-image workflow that ports Linux 0.12 to the repository's QEMU runtime.

## What This Rebuild Produces

- `rebuild/out/images/bootimage-0.12-hd`
  - Built from the patched Linux 0.12 kernel source in `vendor/src/linux-0.12.tar.gz`
- `rebuild/out/images/hdc-0.12.img`
  - Rebuilt locally from the committed canonical rootfs baseline in `rebuild/rootfs/base.tar` plus `rebuild/rootfs/overlay/`

The Linux 0.12 source tarball does not contain a complete userland, so this phase rebuilds the hard disk image from a controlled rootfs baseline instead of claiming that the whole disk image comes from kernel source alone.

## Requirements

- Python 3
- Docker
- `qemu-system-i386` on `PATH` if you want to run `verify` or `promote`

## Commands

Refresh the canonical rootfs baseline from the current default hard disk image:

```sh
python3 rebuild/driver.py capture-rootfs
```

Build the containerized toolchain and regenerate both runtime images:

```sh
python3 rebuild/driver.py build
```

Boot QEMU with the rebuilt images, enter the shell, and run `ls`:

```sh
python3 rebuild/driver.py verify
```

After rebuilt images verify successfully, copy them into `vendor/images/` and re-run the default runtime verification path:

```sh
python3 rebuild/driver.py promote
```

## Typical Workflow

If you only want to reproduce the current source build:

```sh
python3 rebuild/driver.py build
python3 rebuild/driver.py verify
```

If you also want to replace the default runtime images used by `./scripts/run.sh` and `./scripts/verify.sh`:

```sh
python3 rebuild/driver.py promote
```

## Directory Layout

- `driver.py`: host-side CLI for capture, build, verify, and promote
- `container/`: container-side scripts that capture the rootfs and build images
- `patches/linux-0.12/`: QEMU and modern-toolchain compatibility patches applied to Linux 0.12 source
- `rootfs/base.tar`: canonical rootfs baseline captured from the historical runtime disk
- `rootfs/layout.sfdisk`: partition layout for the rebuilt hard disk image
- `rootfs/overlay/`: repo-owned overlay applied on top of the canonical rootfs baseline
- `out/images/`: rebuilt images generated by `build`
- `out/logs/`: build and capture logs
